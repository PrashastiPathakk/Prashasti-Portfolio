# Secure REST API Design: A Practical Guide

As a Software Development Engineer, I've learned that security isn't an afterthought—it's a core requirement. Whether you're building for a startup or an enterprise, your API is the frontline of your application's defense.

## 1. Authentication & Authorization (JWT)

Don't roll your own crypto. Use battle-tested standards like **JWT (JSON Web Tokens)** for stateless authentication.

```javascript
const jwt = require('jsonwebtoken');

const authenticateToken = (req, res, next) => {
  const authHeader = req.headers['authorization'];
  const token = authHeader && authHeader.split(' ')[1];

  if (!token) return res.sendStatus(401);

  jwt.verify(token, process.env.ACCESS_TOKEN_SECRET, (err, user) => {
    if (err) return res.sendStatus(403);
    req.user = user;
    next();
  });
};
```

## 2. Preventing Injection with Data Modeling

While working with **MongoDB**, I always advocate for strong schema validation via **Mongoose**. This prevents malformed data from compromising the database integrity.

```javascript
// Good: Using Mongoose Schema Validation
const userSchema = new mongoose.Schema({
  email: { type: String, required: true, unique: true },
  password: { type: String, required: true, minlength: 8 }
});
```

## 3. Defense in Depth: Rate Limiting & Sanitization

- **Rate Limiting**: Protect your resources from Brute Force and DoS attacks.
- **Input Sanitization**: Use libraries like `express-validator` to ensure incoming data follows the expected schema.
- **Security Headers**: Implement `helmet` to set secure HTTP headers automatically.

## Conclusion

Building secure APIs requires a proactive mindset. By implementing these patterns, you protect both your users' data and your system's integrity.
